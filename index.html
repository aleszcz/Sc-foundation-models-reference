scRNA-seq · Foundation Models · Field Reference

Four ways to feed
a cell to a transformer.

scFoundation, scGPT, Cell2Sentence, and Geneformer all borrow the LLM playbook — pretrain on tens of millions of cells, then reuse. What separates them is a single design choice: how a cell's gene expression is encoded as model input. Everything downstream follows from that.

Geneformer

Corpus-normalized rank order as tokens — magnitudes discarded entirely.

Read across: scFoundation trusts the numbers, scGPT discretizes them, Cell2Sentence verbalizes them, Geneformer keeps only their order. That one decision shapes the architecture, the objective, and what each model is good at.

Model by model

The details that matter for deployment

Continuous values · read-depth aware

scFoundation

The "keep the real numbers" model. Its identity is modeling sequencing depth explicitly rather than hoping scale washes it out.

Find & install

GitHub biomap-research/scFoundation; weights on Zenodo (models.ckpt, the 50M-0.1B model) into model/models/. Needs PyTorch + CUDA, einops, scanpy, local_attention. GPU model.

Design idea

Continuous expression scalars embedded directly (no binning), over the full ~20k-gene transcriptome. Asymmetric split: a heavy vanilla-transformer encoder sees only the ~10% expressed genes; a lighter Performer decoder spans all genes.

Transformation

Align to the 19,264-gene index → append T (target) and S (source) total-count tokens → encoder pools to a cell embedding; decoder yields gene context embeddings. Setting T > S performs read-depth enhancement.

Pipeline

Run get_embedding.py once to extract embeddings, then feed them to a frozen downstream head — DeepCDR (drug response), SCAD (single-cell sensitivity), GEARS (perturbation). Most tasks need no fine-tuning.

Watch out
  • The old online API was discontinued 30 Apr 2024 — use local weights or migrate to aigp.biomap.com.
  • Use the embeddings, not the reconstructed expression values (pretraining loss wasn't driven to zero; high dropout).
  • Read-depth only is harmonized internally; leave batch/modality to downstream methods (BBKNN, SCAD).
Binned values · generative attention

scGPT

The most literally GPT-like, and the most "fine-tune it." Its trick is a generative attention mask that lets a transformer generate over unordered gene sets.

Find & install

GitHub bowang-lab/scGPT or pip install scgpt (poetry is out of sync — use pip). Python ≥3.7. flash-attn is now optional; weights load on CPU, GPU, or flash-attn via the same load_pretrained.

Checkpoints

whole-human (33M cells) is the default. Also continual-pretrained (zero-shot embeddings), pan-cancer (5.7M), and organ models: brain, blood, heart, lung, kidney. Each folder ships args.json, vocab.json, best_model.pt.

Transformation

Normalize → log → HVG → bin into ~50 equidistant bins. Binned value + gene token are embedded and summed; condition tokens (batch / modality / perturbation) and a <cls> token complete the input. Nonzero genes only during pretraining.

Pipeline

Preprocess → tokenize vs vocab.json → load weights → choose zero-shot (pull <cls> embeddings) or fine-tune. Tasks: annotation, batch & multi-omic integration, perturbation, GRN inference, cell generation.

Watch out
  • Use pip install, not poetry.
  • If you want flash-attn speed, historically pin CUDA 11.7 / flash-attn < 1.0.5 — version mismatches are the usual install failure.
  • Default to whole-human unless your data clearly shares context with an organ model.
Text · ordinary LLM

Cell2Sentence

The clever shortcut: turn a cell into a sentence of gene names and use a normal LLM. The only model where "talk to your cells" and "generate a cell from a description" are native.

Find & install

GitHub vandijklab/cell2sentence (older cards point to -ft) or pip install cell2sentence. Docs on readthedocs. Models on Hugging Face under vandijklab.

Checkpoints
  • pythia-160m-c2s — small, one-GPU starter (immune tissue).
  • C2S-Scale-Pythia-1b-pt — 1B, 57M cells.
  • C2S-Scale-Gemma-2-2B / -27B — current flagships.
Transformation

Rank genes high→low, write the names as space-separated text = a "cell sentence." Tasks run as prompts ("Cell sentence: … The cell type is:"). Reverse mapping (rank→expression) is reconstructed via a log-rank/log-expression fit — approximate.

Pipeline

Preprocess → wrap in CSData → load CSModel → prompt for prediction, generation, embeddings, or (C2S-Scale) perturbation, captioning, QA. Custom & multi-cell prompt templates supported (tutorials 7–8).

Watch out
  • Licensing is mixed: code Apache-2.0, HF weights CC-BY-4.0 — but some C2S-Scale mirrors are non-commercial. Check the specific card.
  • Exact magnitudes are lost; sentence length forces a top-k gene cutoff.
Rank value encoding · in silico perturbation

Geneformer

The "ranks only" BERT. Built for small-data fine-tuning and in silico perturbation — asking what happens to cell state when a gene is pushed up or deleted.

Find & install

Lives on Hugging Face, not GitHub: ctheodoris/Geneformer. Needs git-lfs: git lfs install && git clone … && pip install . Docs on readthedocs.

Checkpoints

V1 (2021, ~30M cells, incl. 10M model). V2 (Dec 2024, default): 104M & 316M params, 104M cells, context 4096. Plus V2-104M_CLcancer (continual-tuned on ~14M cancer cells).

Transformation

Rank genes by expression normalized across the whole corpus — so ubiquitous genes sink and cell-state-defining genes rise. Encode as tokens (with [CLS] in V2). Magnitudes discarded; representation is nonparametric.

Pipeline

ENSEMBL-ID your AnnData → TranscriptomeTokenizer → then EmbExtractor (zero-shot embeddings), a fine-tuned classifier head, or InSilicoPerturber (delete = drop from ranks; overexpress = move to front).

Watch out
  • V1 vs V2 configs differ: V1 → special_token=False, input_size=2048; V2 → special_token=True, input_size=4096. Use the matching token dictionary / gene-median file.
  • Requires ENSEMBL gene IDs (no version suffix) in obs["ensembl_id"].
  • CELLxGENE Census ships precomputed embeddings — often faster than running inference.
How to choose

One line each, when you're deciding

Reach for scFoundation

when read depth varies across your data, you want shallow profiles enhanced, or you need strong frozen embeddings with no fine-tuning.

Reach for scGPT

when you want a flexible all-rounder to fine-tune across annotation, integration, perturbation, and GRN tasks from one base.

Reach for Cell2Sentence

when you want natural-language outputs — QA, captioning, cell generation from text — or to unify transcriptomics with literature.

Reach for Geneformer

when you have limited labeled data or your central question is in silico perturbation of cell state and network regulators.
